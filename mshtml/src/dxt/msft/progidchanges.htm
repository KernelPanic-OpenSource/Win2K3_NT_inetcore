When we added our new optimized code to the Trident tree and started testing it, we found that Direct Animation had a bug which prevented optimized transforms from displaying correctly.  VizAct uses DA with DXTrasnforms heavily and we can’t make a fix to DA, so we decided to keep support for these transforms to be un-optimized and create new ProgIDs and CLSIDs that would support optimization.

 

Since we had to create new ProgIDs to do this, we decided this would be a good time to drop the “Cr” (chrome) from the beginning of the DXTransform names that used this prefix.  Below is a list of the old ProgIDs and the corresponding new ProgIDs.  Some CLSIDs also needed to change for DXTransforms affected by the optimization bug.

 

With the IE5.5 release we won’t be documenting the old ProgIDs or CLSIDs.  Any new transforms will support optimization without regard for the DA bug.
